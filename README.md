# 记忆代理人 - 武器锻造台

这是为「记忆代理人 x 落花星雨」服务器制作的专属武器定制网页。玩家可以在这里可视化设计自己的传说武器，系统会自动计算价格并生成配置代码。

## 🎮 玩家使用指南

1. **填写信息**：输入你的游戏 ID，选择想要的武器材质（如钻石剑、下界合金斧等）和自定义名称。
2. **定制属性**：
   - 在左侧选择需要的属性（攻击、防御、特殊效果等）。
   - **计费公式**：
    > 系统采用**平滑递增的凸函数**定价（无阈值、越高越贵），防止数值膨胀；部分属性支持“区间输入”，区间会按凸函数的**期望价格**计费（更科学，区间越宽自然越贵）。此外还有“组合加价”（条数越多越贵）来杜绝分批定制造成的套利。
     
    **最终价格计算（定制模式）：**
     ```
     最终价格 = ceil(max(50, 总价Raw))
     ```
     *(即：总价最低 50 元起步，向上取整)*

    **升级补差价（升级模式）：**
     ```
     补差价 = ceil(max(0, 新总价Raw - 旧总价Raw))
     ```
     *(这保证了“分批升级的总支付 = 一次性定制最终总价”，无法投机取巧。升级不触发 50 起步价。)*

    **单项基础价（baseCostRaw）计算**：
     1. **通用（非区间）属性**：使用“进阶定价曲线”
        ```
        曲线价Raw = val × costFactor × (1 + (val / curveRef) ^ gamma)
        ```
        - `val`：属性数值（百分比属性同样按数值计算，例如 10% => val=10）
        - `costFactor`：属性单价系数（写在属性配置里）
        - `curveRef`：曲线参考点
        - `gamma`：曲线幂指数（目前为 1.0）
     
    2. **支持区间的属性**（目前包括：伤害 / PVE攻击 / PVP攻击 / 真实攻击 / 火焰攻击 / 雷电攻击 / 防御 / 生命恢复）
        - 输入格式示例：`1000-2000` / `1000~2000` / `1000～2000`
        - **科学定价**：把区间理解为“数值在区间内均匀波动/随机”（\(V \sim U(min, max)\)），按期望价格计费：
          ```
          曲线价Raw = E[ val × costFactor × (1 + (val / curveRef) ^ gamma) ] ,  val ~ Uniform(min, max)
          ```
        - *(直观理解：在凸函数下，区间越宽，不确定性越大，期望价格自然更高；无需额外手工宽度惩罚。)*

    3. **部位倍率（slotMultiplier）**
       - 主手/头盔/衣服/裤子/鞋子：`1.0`
       - 副手：`2.0`
       - 计算：
         ```
         baseCostRaw = 曲线价Raw × slotMultiplier
         ```

    4. **组合加价（阶梯倍率，按 baseCostRaw 从高到低排序）**
       - 目的：**条数越多越贵**，并且升级用“新总价-旧总价”补差后，能保证分批与一次性最终总价一致（无套利）。
       - 做法：把所有已选属性的 `baseCostRaw` **从高到低排序**，排名从 0 开始：
         ```
         tierMultiplier(rank) = min(1 + COMBO_TIER_STEP * rank, COMBO_MULTIPLIER_CAP)
         单项最终价Raw = baseCostRaw × tierMultiplier(rank)
         总价Raw = ∑ 单项最终价Raw
         ```
       - 直观效果：
         - 第 1 条（rank=0）×1
         - 第 2 条（rank=1）×(1 + step)
         - 第 3 条（rank=2）×(1 + 2·step)
         - ……
       - 这样“新增一个很小的属性”通常排在末尾，只会让末尾那一条更贵，不会导致大词条的倍率突然跳变。
     
     **参数示例**：
     - **部位倍率**：副手 = 2.0，其他部位 = 1.0
     - **最低消费**：50 元

     **显示取整规则（保证每条显示相加 = 总价）**：
     - 页面展示每条属性旁边的 `+xxx` 为整数，为避免“分别 ceil 后相加 ≠ 总价”的误差，系统会对各条 raw 费用做一次统一的取整分配，使：
       - 定制模式：所有 `+xxx` 相加 = `ceil(总价Raw)`（若触发 50 起步价，会额外显示 `起步价补差 +X`）
       - 升级模式：所有 `+xxx` 相加 = `ceil(补差价Raw)`
3. **提交订单**：
   - 满意后点击右侧的 **「复制配置代码」** 按钮。
   - 将复制的内容直接发送给服主进行实装。
